YouTube API Key
The YouTube API key is used to query your YouTube channel to examine your playlist to determine the most recently recorded livestream. You must create this key via the Google Cloud console.
Channel ID
The YouTube channel ID is used to look for your active livestream and to discover and explore your playlists. You can find your channel ID via the You will need to get the channel ID from your advanced YouTube account settings.
Playlist ID
You can find your playlist ID by doing the following:
  • Go to Youtube
  • Sign in (upper right corner) if not already logged in
  • Select "Your Channel" under the user icon in the upper right corner.
  • Click on "Playlists"
  • Click on the "View full playlist" under the playlist you want to use.
  • Look at the url. You will see the playlist ID following "list="
Transition To Upcoming Livestream
The transition setting determines how far in advance of an upcoming livestream your website will switch from embedding the most recently recorded livestream to embedding the upcoming livestream. If left blank, the transition will occur at the scheduled start time.

Embedded Livestream Options
There are a number of options that can be configured when embedding a YouTube video within a webpage. For more information on each of these, see Player Parameters in the Youtube developer documentaiton
  • > autoplay playback will occur without any user interaction with the player
  • > controls playback controls are displayed in the player
  • > disable keyboard control of playback using the keyboard is disabled
  • > full screen allow the user to display the livestream full screen
  • > modest branding suppress the YouTube logo from the livestream unless paused
YouTube Query Frequency
YouTube API queries are subject to a quota system, allowing only so many queries per day. If you are using the API key specified above for more than just this plugin, you may wish to reduce the query frequency to avoid hitting quota limitations. Otherwise, the default limit of no more than 1 query per minute should be quite sufficient to avoid quota issues.
